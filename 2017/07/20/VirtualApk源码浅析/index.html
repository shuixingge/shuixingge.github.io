如果对Activity启动流程比较熟悉的时候，可以知道startAcitvity一定会走到Instrumentation的execStartActivity,由于选了hook了Instrumentation,所以会走到,VAInstrumentation.execStartActivity();

  • transformIntentToExplicitAsNeeded: 当activity为隐式启动时，其component为空,这个方法的作用就是根据intent设置的action,category等信息去插件中匹配中的acitivity，然后设置component，简单说就是将隐式启动转为显示启动。

public void markIntentIfNeeded(Intent intent) {
        if (intent.getComponent() == null) {
            return;
        }

        String targetPackageName = intent.getComponent().getPackageName();
        String targetClassName = intent.getComponent().getClassName();
        // search map and return specific launchmode stub activity
        if (!targetPackageName.equals(mContext.getPackageName()) && mPluginManager.getLoadedPlugin(targetPackageName) != null) {
            intent.putExtra(Constants.KEY_IS_PLUGIN, true);
            intent.putExtra(Constants.KEY_TARGET_PACKAGE, targetPackageName);
            intent.putExtra(Constants.KEY_TARGET_ACTIVITY, targetClassName);
            dispatchStubActivity(intent);
        }
    }

markIntentIfNeeded()将原始的intent(启动的目标插件)的信息保存一下，然后调用dispatchStubActivity匹配到宿主中对应的“代理”Activity。
dispatchStubActivity:根据启动模式去匹配宿主中预先在AndroidManifest 文件中占坑的代理Activity.并将intent待启动的activity置为代理activity。

1.2 还原插件Activity。

还是需要对一定activity启动流程有一定了解，activity启动最终还是会回到ActivityThread,即 ActivityThread#handleLaunchActivity->mInstrumentation.newActivity()。 所有最终会回到VAInstrumentation的newActivity(), 代码如下

@Override
   public Activity newActivity(ClassLoader cl, String className, Intent intent) throws InstantiationException, IllegalAccessException, ClassNotFoundException {
       try {
           cl.loadClass(className);
       } catch (ClassNotFoundException e) {
           LoadedPlugin plugin = this.mPluginManager.getLoadedPlugin(intent);
           String targetClassName = PluginUtil.getTargetActivity(intent);

           Log.i(TAG, String.format("newActivity[%s : %s]", className, targetClassName));

           if (targetClassName != null) {
               Activity activity = mBase.newActivity(plugin.getClassLoader(), targetClassName, intent);
               activity.setIntent(intent);

               try {
                   // for 4.1+
                   ReflectUtil.setField(ContextThemeWrapper.class, activity, "mResources", plugin.getResources());
               } catch (Exception ignored) {
                   // ignored.
               }

               return activity;
           }
       }

       return mBase.newActivity(cl, className, intent);
   }
