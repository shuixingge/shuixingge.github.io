1.Activity原理

采用宿主manifest中占坑的方式来绕过系统校验，然后再加载真正的activity

1.1 占坑

这里选择hook了Instrumentation，具体做法是先通过反射得到ActivityThread中的Instrumentation的变量。然后以此构建代理对象VAInstrumentation, 然后把此设为ActivityThread的mInstrumentation变量。具体实现代码如下

如果对Activity启动流程比较熟悉的时候，可以知道startAcitvity一定会走到Instrumentation的execStartActivity,由于选了hook了Instrumentation,所以会走到,VAInstrumentation.execStartActivity();

  • transformIntentToExplicitAsNeeded: 当activity为隐式启动时，其component为空,这个方法的作用就是根据intent设置的action,category等信息去插件中匹配中的acitivity，然后设置component，简单说就是将隐式启动转为显示启动。

    private void hookSystemServices() {

    try {
    Singleton<IActivityManager> defaultSingleton = (Singleton<IActivityManager>) ReflectUtil.getField(ActivityManagerNative.class, null, "gDefault");
    IActivityManager activityManagerProxy = ActivityManagerProxy.newInstance(this, defaultSingleton.get());

    // Hook IActivityManager from ActivityManagerNative
    ReflectUtil.setField(defaultSingleton.getClass().getSuperclass(), defaultSingleton, "mInstance", activityManagerProxy);

    if (defaultSingleton.get() == activityManagerProxy) {
        this.mActivityManager = activityManagerProxy;
    }
} catch (Exception e) {
    e.printStackTrace();
}

    }

    但是由于我们hook了ActivityManagerProxy,所以会调用invoke方法，调用startService方法。

startService： 如果ResolveInfo为空则说明这个Service不是插件中Service,则直接调原本的ActivityManager的对象的startService方法。否则调用startDelegateServiceForTarget

startDelegateServiceForTarge: wrapperTargetIntent主要Intent 做了处理。然后直接启动Service。

**wrapperTargetIntent:** 把intent设置为Component的target设置为插件的service。其中当多进程时，delegate为RemoteService。同一进程时delegate为LocalService.

#####2.2 Service代理分发

主要通过插件的ClassLoader去加载插件的Service class。然后通过反射创建出实例对象,
然后反射调用了Service的attach方法，最后手动调用onCreate方法，onStartCommand 方法。
